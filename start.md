# WHAT YOU'RE BUILDING

An autonomous, self-improving business analysis system inspired by Karpathy's autoresearch. The user has course materials from a Master of Entrepreneurship & Innovation program (Design Thinking, Product Management, Innovation Management, Advanced Strategy, Entrepreneurship). The system:

1. Extracts knowledge from course transcripts into distilled 1-2 page docs
2. Creates AI advisor personas from those docs — each with a unique personality
3. When given a business idea, runs it through ALL advisors sequentially (waterfall, each sees everything before them)
4. Each advisor uses web search (Brave MCP) for real market data
5. A critic agent scores the analysis on 8 dimensions (0-100)
6. The system LOOPS: focuses on weak dimensions, advisors re-analyze, critic re-scores
7. Each improvement is git-committed (like autoresearch commits better models)
8. Converges when score plateaus or hits target (85/100)
9. A live HTML dashboard shows convergence in real-time (reads state.json)
10. Runs fully autonomously with --dangerously-skip-permissions

The user runs this on cheap models via OpenRouter (~$0.50/analysis) to preserve their Claude Max subscription tokens.

## FILE 3: run-loop.sh (~350 lines)

The auto-regressive convergence loop. This is the heart of the system.

**Arguments:**
```bash
./run-loop.sh "Business idea text"              # idea as text
./run-loop.sh ideas/my-idea.md                   # idea from file
./run-loop.sh ideas/my-idea.md --max-rounds 8    # custom rounds
./run-loop.sh ideas/my-idea.md --budget 5.00     # cost limit
```

**Variables:**
```bash
MAX_ROUNDS=8
BUDGET_USD=5.00
CONVERGENCE_THRESHOLD=2      # stop if improvement < this
MIN_SCORE=85                  # target score to stop early
```

**The loop logic (CRITICAL — implement this exactly):**

```
initialize:
  - create output/run-[timestamp]/ directory tree
  - git init if needed
  - create feature branch: analysis-[idea-name]-[timestamp]
  - write initial state.json

for round in 1..MAX_ROUNDS:
  
  write state.json → {phase: "analyzing", round: N, status: "running"}
  
  if round == 1:
    prompt = full analysis from scratch
    - include all advisor personas
    - include all knowledge docs
    - include business idea
    - instruction: "save each advisor output separately, then combined report, then score"
  else:
    prompt = targeted improvement
    - include business idea
    - include critic's summary_for_next_round from round N-1 (NOT full reports)
    - include critic's gaps and next_round_focus
    - instruction: "focus on the weak dimensions, do more web research, merge improvements into previous report"
  
  run: claude -p "$prompt" --dangerously-skip-permissions
  
  read score.json from output:
    - parse overall_score using python3 -c "import json; ..."
    - parse weakest_dimension
    - handle: if score.json missing or invalid → score = 0, retry once
  
  update scores/history.json (append this round's scores)
  update improvements.json (what improved)
  write state.json → {phase: "scored", score: X, round: N}
  
  convergence checks:
    if score >= MIN_SCORE → "TARGET REACHED", break
    if round > 2 AND improvement < CONVERGENCE_THRESHOLD → "CONVERGED", break
    if score < previous_score → "REGRESSION, keeping previous", note but continue
  
  git add -A && git commit -m "Round N: score X/100 (+Y) — improved Z"
  
  PREV_SCORE = CURRENT_SCORE

end loop

copy best round's report to FINAL-REPORT-score-XX.md
write final state.json → {status: "complete"}
print summary
```

**state.json format (dashboard reads this):**
```json
{
  "run_id": "2026-03-10_14-30-00",
  "idea": "first 200 chars of idea...",
  "current_round": 3,
  "max_rounds": 8,
  "phase": "analyzing",
  "active_agent": "strategy-advisor",
  "current_score": 67,
  "target_score": 85,
  "convergence_threshold": 2,
  "status": "running",
  "detail": "Round 3 — strategy advisor analyzing with web research",
  "budget_max_usd": 5.00,
  "timestamp": "2026-03-10T14:35:22Z",
  "scores_history": [
    {"round": 1, "overall_score": 42, "dimensions": {...}},
    {"round": 2, "overall_score": 58, "dimensions": {...}}
  ],
  "improvements": [
    {"round": 2, "score": 58, "improvement": 16, "focus": "market research"}
  ]
}
```

## FILE 5: extract_standalone.py (~300 lines)

Standalone Python script for knowledge extraction using ANY LLM provider.
Bypasses Claude Code entirely — saves Max tokens.

**Supported providers:**
- openrouter (default) → https://openrouter.ai/api/v1/chat/completions
- deepseek → https://api.deepseek.com/chat/completions
- anthropic → https://api.anthropic.com/v1/messages (different format!)
- openai → https://api.openai.com/v1/chat/completions
- ollama → http://localhost:11434/v1/chat/completions

**Environment variables:**
```bash
LLM_PROVIDER=openrouter
LLM_API_KEY=sk-or-v1-xxx
LLM_MODEL=google/gemini-2.5-flash
```

**What it does:**
1. Scans courses/ for subfolders
2. For each course, reads all .md/.txt/.html/.csv/.json files
3. Concatenates content (truncates at 100K chars if needed)
4. Calls LLM with extraction prompt → saves to knowledge/[name].md
5. Calls LLM with persona generation prompt → saves to .claude/agents/[NN]-[name]-advisor.md
6. Creates the orchestrator and critic personas
7. Prints progress with emoji indicators

**Error handling:**
- API failures → print error, skip course, continue
- File read errors → log warning, skip file, continue
- Timeout → 120 second timeout per API call
- Provider-specific: Anthropic uses different request format (messages API with x-api-key header, response in data.content[0].text)

**Requirements:** Only `requests` library (pip install requests)
